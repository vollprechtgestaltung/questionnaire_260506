## Entscheidungen

- **Stack-Profil des Vorprojekts wird übernommen** (Svelte/Vite/Supabase/
  Vercel/PWA, Plain JS). `docs/stack.md` muss nächste Session entsprechend
  korrigiert werden — aktuelle Werte (TS strict + biome) sind veraltet.
- **Plain JS statt TS strict** — bewusster Override des Senior-Defaults,
  weil das Vorprojekt fertig in JS vorliegt; TS-Migration wäre eigenes
  Workpaket. → muss als ADR in `docs/decisions.md` festgehalten werden.
- **Supabase- und Vercel-Instanz des Vorprojekts werden reused** —
  Edge Function bleibt mit unverändertem `ALLOWED_ORIGIN`, kein
  Redeploy nötig.
- **Git-History via Graft erhalten**, nicht force-push. Vorprojekt-`.git/`
  wird über die frische `.git/` kopiert, danach ein neuer Commit on top —
  lineare History, normale Push.
- **`.env` kopiert der User von Hand** — nie via Copy-Tool.
- **Network-Allowlist** (`*.supabase.co`, `*.vercel.app` etc.) erst nach
  expliziter User-Freigabe in `.claude/settings.json` einfügen.
