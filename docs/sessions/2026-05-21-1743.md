## Kontext

Analyse der Offline-Queue-Robustheit für ein 3-iPad-Messe-Szenario: 3h Netzwerkausfall, 814 Votes akkumuliert, simultaner Reconnect. Daraus folgte ein konkreter Fix am Rate Limiting, ein Security-Audit, und die Klärung des Production-Status.

## Was passiert ist

**Analyse: Rate-Limit-Problem im Queue-Drain**
- Problem identifiziert: `flushQueue()` verarbeitet 1 Vote pro 2.5s-Poll-Zyklus; Rate Limit (15s/device_id) blockiert alle Queued Votes — iPad C mit 426 Votes würde 107 Minuten brauchen
- Ursache: Rate-Limit-Check prüft `created_at` (= Flush-Zeitpunkt), nicht den echten Abstimmungszeitpunkt

**Fix: `voted_at` + kombinierte Rate-Limit-Logik**
- DB: `votes.voted_at timestamptz NULL` via Migration hinzugefügt
- `VoteScreen.svelte`: `voted_at: new Date().toISOString()` im Vote-Objekt
- `vote.js flushQueue`: `queued: true` im Payload
- Edge Function v8: Live-Votes prüfen weiterhin `created_at` (Rückwärtskompatibilität); Queue-Flushes prüfen `voted_at ±15s` — beurteilt Vote gegen originalen Abstimmungszeitpunkt
- Fallback auf `now()` für pre-Migration Queue-Einträge ohne `voted_at`
- Drain-Zeit nach Fix: ~18 min für 426 Votes (statt 107 min)

**Security-Audit**
- 0 Vulnerabilities (`npm audit`), keine Secrets im Code, `.env` nicht getrackt
- Sandbox aktiv, Hook-Tests grün (55/55)
- RLS korrekt: nur SELECT für Anon-Key, INSERT nur via Service Role Key durch Edge Function
- WARNUNG behoben: CSP-Header in `vercel.json` ergänzt (`connect-src Supabase`, `frame-ancestors 'none'`)
- INFO behoben: `error.message`-Leak in `heartbeat.js` → `'internal_error'`
- CRON_SECRET in Vercel gesetzt → heartbeat-Schutz aktiv

## Entscheidungen

- **`voted_at` in DB speichern** — ermöglicht korrekte Rate-Limit-Prüfung bei Queue-Drain und saubere historische Timestamps
- **Live-Votes: `created_at`-Check beibehalten** — Rückwärtskompatibilität mit pre-Migration Rows
- **`queued: true` Flag akzeptiert** — Client-seitig setzbar, aber UNIQUE Constraint verhindert Duplikate; kein Sicherheitsproblem
- **Production-Anspruch gilt** — „Prototyp" aus CLAUDE.md ist historischer Auftragsrahmen, nicht Qualitäts-Entschuldigung; vollprecht-Workflow: Prototyp solide bauen → direkt in Production überführen
